# AGENTS.md

このリポジトリで作業する AI エージェント向けの運用ルールです。

## 目的
- 変更の再現性・安全性・レビュー容易性を高める。

## 作業前
- 影響範囲を明確化し、変更対象・非対象を最初に短く宣言する。
- 仕様が曖昧な場合は、推測実装を避け、前提・解釈を明示してから進める。

## 実装ルール
- 変更は最小差分を原則とし、無関係なリファクタ・整形を混在させない。
- 既存の設計意図（命名規約、責務分離、依存方向）を優先し、逸脱する場合は理由を残す。
- エラー文言・ログ文言を変更する際は、運用時の検索性を下げない一貫した語彙を使う。

## テストと品質基準
- 実装時は、必ずテストも同時に更新し、**テストカバレッジ 100% を維持**すること。
- バグ修正では、再発防止の回帰テストを先に追加してから修正する。
- 失敗系（異常系・境界値）のテストを省略しない。

## 変更説明
- コミットメッセージは「何を」ではなく「なぜ」を含める。
- PR では以下を必須記載とする：
  - 背景 / 目的
  - 主な変更点
  - テスト観点（正常系・異常系）
  - 互換性影響（あれば）

## 禁止事項
- 機密情報・個人情報・トークン類をログ/コード/履歴に残さない。
- 根拠のない性能改善主張や、未検証の断定表現をしない。

## 迷ったとき
- 判断に迷う場合は、可逆で安全な方針を選び、トレードオフを明記する。
